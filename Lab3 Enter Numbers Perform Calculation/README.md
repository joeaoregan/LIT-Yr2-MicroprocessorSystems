### Lab 3 – Code a Division, Addition and multiplication calculators and Prime number debugging

In this lab you will write a number of 68000 programs. 

The first program will read in two numbers. The Numerator (the number above the line ) and the (denominator) below the line. The program will divide the Numerator by the divisor e.g. 15/4.

The second part will read in 4 numbers and perform a calculation.

In the third part of the lab you will load a prime number program and debug/trace the programs execution and explain how it works. Debugging and tracing a program are essential skills for a software developer.

1. – The program should store the Numerator in D4 as a word .w
2. – The program should store the Denominator in D3 as a word
3. – The answer should provide the Quotient and the Remainder as base 10 numbers and printed to the screen e.g. The Quotient is 3 the remainder is 5
4. – Division stores the quotient in the lower 16 bits and the remainder in the upper 16 bits. This means you will have to move out the remainder to a temp location and swap the contents of the Data Register. E.g SWAP D1
5. – Test the program and demonstrate it to your lecturer. Document the code and include the code and the description of the program in your lab report
6. – Write a program to read in 4 numbers. The numbers can be stored in Data registers or defined storage locations A,B, C & D. The program should perform the following calculation  
    (A+B)*(C+D)
7. – Demonstrate the program to your lecturer
8. – Download the attached program from the Moodle page. In your lab report describe how the program determines if the number is a prime. Use the two following numbers for your explanation.          
    - Debug using the number 7
    - Debug using the number 8
9. – As part of the debug record in your lab book the values of all registers and variables as it steps through the program. Use F7 to step through the program line by line.
10. – Lab report must include and intro and conclusion.
